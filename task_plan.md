# Task Plan: PiliPlus 功能迁移规划

## Goal
梳理参考项目 `PiliPlus` 的能力边界，结合当前 `bilibili_webos` 的真实基线，产出一份可长期执行的 WebOS 端功能迁移计划文档。

## Phases
- [x] Phase 1: 审视当前 WebOS 仓库结构与已有能力
- [x] Phase 2: 调研参考项目的功能地图、入口结构与关键模块
- [x] Phase 3: 设计适合 TV 场景的迁移分期与模块映射
- [x] Phase 4: 编写正式计划文档并同步研究笔记
- [ ] Phase 5: 按计划逐阶段实施与验收

## Key Questions
1. `PiliPlus` 中哪些能力属于 WebOS MVP 的主路径，应该优先迁移？
2. 哪些能力明显带有移动端/桌面端特征，不适合早期照搬到 TV？
3. 当前 `bilibili_webos` 缺的首先是页面，还是更底层的路由、数据、播放器、鉴权与焦点基础设施？
4. 如何把“照着抄”变成可执行的里程碑，而不是失控的大而全需求？

## Decisions Made
- 不按 `PiliPlus/lib/pages/*` 一页页机械移植，而按业务域和 TV 关键路径拆分阶段。
- 迁移优先级遵循：播放闭环 > 遥控器可用 > 焦点稳定 > 内容资产 > 社交长尾功能。
- 先覆盖首页、搜索、详情/播放器、登录、历史、收藏/稍后再看、个人中心等 TV 主路径。
- 动态、私信、发布、DLNA、离线缓存、复杂设置等重功能先后置，不在首批闭环内承诺。

## Errors Encountered
- 使用 `rg` 搜索 `video` 目录能力点时，PowerShell 将 `|` 解释为管道，导致命令失败；后续改为更小粒度的文件定向读取规避。

## Status
**Currently in Phase 5** - 迁移规划文档已写入，下一步可按文档从首页数据化与路由扩展开始实施。
