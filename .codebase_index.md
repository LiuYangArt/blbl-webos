# `.codebase_index.md`

## 1. 仓库概览

- 项目名：`bilibili-webos`
- 目标平台：`LG webOS TV`
- 前端栈：`React 19 + TypeScript 5 + Vite 7`
- 当前阶段：`MVP UI 骨架 + 遥控器焦点导航 + webOS 打包/安装脚本`
- 设计基线：默认按 `1920x1080` 设计，遥控器 5-way 导航优先
- 重点约束：
  - 业务组件里不要直接使用 `window.webOS` / `PalmSystem` / 原始按键码
  - 焦点流、返回链路、播放器基础操作优先级高于视觉细节
  - webOS 真机打包/安装/启动优先走 `scripts/webos-cli.mjs`

## 4. 关键入口与调用关系

### 4.1 运行入口

1. `index.html`
   - 挂载 `#root`
   - 先注入 `public/webOSTV.js`
   - 再加载 `src/main.tsx`
2. `src/main.tsx`
   - 挂载 React 根节点
   - 注入全局样式 `src/styles/app.css`
3. `src/App.tsx`
   - 组装应用壳
   - 维护页面栈
   - 挂载遥控器监听
   - 页面切换后重新聚焦首个可聚焦元素

### 4.2 页面切换

- `src/app/usePageStack.ts`
  - 一个轻量页面栈 Hook
  - 当前只管理两类页面：
    - `home`
    - `player`
- `src/App.tsx`
  - 首页通过 `onOpenPlayer(title)` 推入播放器页
  - 返回时优先 `pop()`，栈空则走平台返回

### 4.3 遥控器与焦点

- `src/platform/remote.ts`
  - 统一监听 `keydown`
  - 将方向键映射到 `moveFocus`
  - 将确认键映射到 `activateFocused`
  - 将返回键交给上层 `onBack`
- `src/platform/focus.ts`
  - 使用 `data-focus-row` / `data-focus-col` 做规则化焦点移动
  - 当前策略是按同行/同列最近元素查找，不是空间几何导航
- `src/components/FocusButton.tsx`
  - 所有主要可操作元素的基础组件
  - 负责写入焦点坐标数据

### 4.4 webOS 平台边界

- `src/platform/webos.ts`
  - 统一定义遥控器 key code
  - 封装 `platformBack`
  - 提供 `isWebOSAvailable`、`getKeyboardVisible`、`readDeviceInfo`
- `src/types/webos.d.ts`
  - 为 `window.webOS` / `PalmSystem` 提供全局类型

## 6. 配置与构建文件

    - `npm run lint`

  - 开启 `sourcemap`

    - version：`0.1.0`
    - resolution：`1920x1080`
    - `disableBackHistoryAPI: true`
    - `supportTouchMode: "virtual"`

## 10. 当前代码库的几个明显特点

- 代码量还不大，核心逻辑主要集中在 `src/App.tsx`、`src/platform/*`、`src/styles/app.css`
- 样式是单文件集中管理，改动 UI 时要留意不要把页面专属规则继续堆大
- 焦点系统是 MVP 级实现，适合当前两个页面，但后续复杂页面大概率要升级
- 组件层已经开始成型，后续新增页面应优先复用现有组件而不是直接写页面专属按钮/卡片
- 真机部署链路已经有封装，后续不要绕开 `scripts/webos-cli.mjs` 直接调用裸 LG CLI
