# `.codebase_index.md`

## 1. 仓库概览

- 项目名：`bilibili-webos`
- 目标平台：`LG webOS TV`
- 前端栈：`React 19 + TypeScript 5 + Vite 7`
- 当前阶段：`首页/播放器/搜索/详情/个人页等主链路已接通，持续做 TV 端交互与稳定性优化`
- 测试基线：`Vitest + jsdom`，已覆盖启动参数、缓存、接口封装、播放器编解码、列表 loading gate 等关键逻辑
- 设计基线：默认按 `1920x1080` 设计，遥控器 5-way 导航优先
- 重点约束：
  - 业务组件里不要直接使用 `window.webOS` / `PalmSystem` / 原始按键码
  - 焦点流、返回链路、播放器基础操作优先级高于视觉细节
  - webOS 真机打包/安装/启动优先走 `scripts/webos-cli.mjs`

## 4. 关键入口与调用关系

### 4.1 运行入口

1. `index.html`
   - 挂载 `#root`
   - 先注入 `public/webOSTV.js`
   - 再加载 `src/main.tsx`
2. `src/main.tsx`
   - 挂载 React 根节点
   - 注入全局样式 `src/styles/app.css`
3. `src/App.tsx`
   - 组装应用壳
   - 维护页面栈
   - 挂载遥控器监听
   - 页面切换后重新聚焦首个可聚焦元素

### 4.2 页面切换

- `src/app/usePageStack.ts`
  - 一个轻量页面栈 Hook，承载全局路由 push/replace/pop
  - 当前主路由已覆盖：
    - `home / following / subscriptions / hot / search / search-results`
    - `video-detail / pgc-detail / player / author-space`
    - `history / profile / login / later / favorites / ui-debug`
- `src/App.tsx`
  - 负责所有路由页面装配和页面间跳转
  - 返回时优先 `pop()`，栈空则走平台返回

### 4.3 列表页 Loading Gate

- `src/features/shared/useVideoListPageLoading.ts`
  - 统一组合“数据 ready + 图片 ready + 最短 loading 时长”门控
- `src/features/shared/useImageReadyGate.ts`
  - 首屏关键图预加载就绪判断
- `src/features/shared/useVideoListLoadingGate.ts`
  - 控制 loading 显示/收起时机，避免闪烁
- `src/app/RouteLoadingOverlay.tsx`
  - 在 `AppShell` 层管理右侧主内容区 overlay 显隐

### 4.4 遥控器与焦点

- `src/platform/remote.ts`
  - 统一监听 `keydown`
  - 将方向键映射到 `moveFocus`
  - 将确认键映射到 `activateFocused`
  - 将返回键交给上层 `onBack`
- `src/platform/focus/index.ts`
  - 焦点模块已拆分到 `src/platform/focus/*`
  - 主要包括 `engine.ts`（移动引擎）与 `FocusSection.tsx`（区段容器）
- `src/components/FocusButton.tsx`
  - 所有主要可操作元素的基础组件
  - 负责写入焦点坐标数据

### 4.5 webOS 平台边界

- `src/platform/webos.ts`
  - 统一定义遥控器 key code
  - 封装 `platformBack`
  - 提供 `isWebOSAvailable`、`getKeyboardVisible`、`readDeviceInfo`
- `src/types/webos.d.ts`
  - 为 `window.webOS` / `PalmSystem` 提供全局类型

## 10. 当前代码库的几个明显特点

- 页面与功能覆盖面已经较大，`src/features/*`、`src/services/*`、`src/platform/focus/*` 是排查主干
- 样式仍以 `src/styles/app.css` 为核心，改 UI 时要优先复用 token 与既有组件语义
- 焦点系统已从单文件升级为模块化引擎，复杂焦点问题优先从 `engine.ts` 入手
- 列表页加载体验依赖统一 loading gate，涉及首屏闪烁问题优先看 shared gate hooks + route overlay
- 真机部署链路已固化为脚本化串行流程，优先 `webos:deploy` + `webos:verify-install`
