# 2026-03-26 账号入口下沉、主内容区去壳与播放器设置收敛 Postmortem

## 1. 背景

本轮从 `issue #8` 的真机 UI 重叠问题出发，最终没有选择只修 `z-index` 或局部间距，而是顺手重构了三块信息架构：

1. 账号入口从右上角移到左侧导航底部
2. 常规页面右侧主内容区取消统一大圆角深色壳
3. 播放器设置抽屉从“调试信息堆叠”收敛为“操作优先、状态最小化”

用户目标非常明确：

- 右上角不要再有悬浮账号胶囊
- 右侧内容区不要再套一层大面板
- 播放设置只保留真正有用的信息

## 2. 现象与问题

### 2.1 右上角账号胶囊不适合当前 TV 结构

- 账号胶囊放在右上角时，天然会和右侧内容层争夺顶部空间
- 一旦页面存在内容壳、loading overlay 或局部大面板，右上角入口就容易变成额外层级负担
- 即使单次修好了遮挡，整体结构仍然别扭

### 2.2 右侧内容区的“总壳”让页面层级过重

- 之前常规页面是“左侧导航 + 右侧大壳 + 壳内内容”
- 用户主观感受是页面被框在一个大面板里，上下左右都留着明显安全边距
- 真正去掉外壳后，仍然会残留 `AppShell` 级别的 padding，导致视觉上看起来“还没顶满”

### 2.3 播放设置里最容易犯的错不是缺信息，而是重复信息

- 上面已经有画质和编码按钮
- 如果下面再写“请求画质”“编码偏好”“当前执行策略”，本质上是在重复同一层语义
- 真正需要保留的，是按钮表达不出来、但用户又需要知道的实际结果

## 3. 根因分析

### 3.1 “问题出在 topbar”只是表层，根因是入口放错了层级

- 账号入口本质上属于“全局导航/账户入口”，不是“页面内容区角标”
- 把它放在右上角，会逼它长期和页面内容、overlay、壳容器一起竞争版面
- 这不是一个纯 CSS 层级问题，而是信息架构位置问题

### 3.2 “去壳”之后还没顶满，是因为壳不只存在于 `page-shell`

- 首轮去掉了页面级 `page-shell` 的背景、圆角和阴影
- 但 `tv-app-main` 仍保留了上下 padding
- 所以视觉结果变成：内容壳没了，但页面主舞台仍被 AppShell 的安全边距框住

结论：

- 要让右侧内容区真正“顶满”，必须同时检查页面容器层和 AppShell 层，不能只改页面组件自己的壳

### 3.3 左侧导航图标展开后左移，根因是对齐策略切换了

- 收起态图标是居中对齐
- 展开态图标切成了左对齐
- 于是用户会看到图标在展开瞬间发生横向位移

结论：

- 侧栏展开/收起时，图标基线必须保持固定
- 变化只能发生在“文本展开出来”，不能发生在“图标自己挪位置”

### 3.4 播放设置的正确拆法是“用户意图”和“实际结果”分层

- 画质按钮、编码按钮负责表达用户当前选择
- 状态区只负责表达“现在实际播成了什么”
- 如果状态区开始重复“用户刚刚选过什么”，页面很快又会膨胀回调试面板

## 4. 本轮最终决策

### 4.1 账号入口

- 删除右上角账号胶囊
- 把账户位固定到左侧导航底部
- 收起态只显示头像
- 展开态显示头像和用户名；未登录时显示“游客模式”

### 4.2 常规页面主舞台

- 去掉常规页面统一深色大壳
- 去掉右上角 topbar
- 继续保留页面内部真实组件的局部表面
- 同时去掉 AppShell 对右侧主舞台造成的上下多余留白

### 4.3 播放设置

- 默认页只保留：
  - 画质按钮组
  - 编码按钮组
  - 1 条当前播放结果摘要
  - 少量快捷操作
- 调试信息不回流到默认页
- 只额外保留 `BV号` 作为截图排障锚点

## 5. 已确认有效的经验

1. TV 端账号入口优先跟导航走，不要漂在页面右上角做“角标式入口”
2. 视觉上想“去壳”，要同时检查页面容器和应用壳容器；只改页面组件经常会留下假去壳
3. 左侧导航展开/收起时，允许文本展开，不允许图标横向漂移
4. 播放设置里最重要的不是“尽量多留信息”，而是“避免重复表达同一层语义”
5. 如果需要给用户留一个最小排障锚点，优先保留 `BV号` 这类截图可用信息，而不是整套工程调试字段
6. 用户已经明确过“不要冗余”时，不要再用“也许以后会有用”给默认页加回更多字段

## 6. 可复用规则

后续再做 TV 面板、抽屉或设置页时，默认先套用下面三条：

1. 按钮表达“用户意图”
2. 状态区表达“实际结果”
3. 调试信息默认不进主面板

如果实现后页面密度超出这个边界，优先删字段，不优先加折叠层。

## 7. 后续行动

1. 继续观察首页、热门、历史、个人页在“去壳”后的整体视觉密度，确认不会因为少了总壳而变得过散
2. 如果播放器设置后续还要补额外排障能力，优先考虑单独的 debug 路径，而不是继续把默认抽屉撑大
3. 真机继续验证左侧导航账户位在不同登录状态下的头像加载、焦点流和返回链路
