# 账号入口下沉与主内容区减层方案

## 1. 背景

当前 UI 有两个明显问题：

- 右上角账号胶囊悬浮在主内容区之上，容易与页面内容层级冲突，也让页面右上角显得拥挤。
- 常规页面右侧主内容区外面包了一层大圆角深色底板，内部又叠标题、频道条、卡片等层级，视觉上偏重，不够直接。
- 播放设置抽屉里混入了过多偏调试性质的信息，用户真正常用的设置与状态信息没有被放在第一优先级。

本方案的目标不是局部补丁，而是顺手把信息架构收干净：把账号入口并入左侧导航，把右侧主舞台从“大面板套内容”改为“内容直接呈现”，并把播放设置抽屉压缩成真正适合 TV 端操作的信息密度。

## 2. 本次已确认的设计决策

### 2.1 左侧导航底部账户位

- 删除右上角账号入口，不再保留 `TopbarProfilePill` 在主内容区顶部的悬浮呈现。
- 在左侧导航栏最底部新增一个固定账户位，作为导航的一部分存在。
- 账户位固定贴近左侧导航底部区域，不跟随右侧页面滚动。
- 账户位和上方导航按钮使用同一套对齐基线、同一套焦点语义、同一套收起/展开机制。

### 2.2 收起态与展开态

- 导航收起时，账户位只显示头像入口。
- 该头像入口的尺寸、垂直节奏、左右对齐方式必须和普通导航按钮图标位保持一致，看起来像导航列表自然延伸出来的最后一项。
- 导航展开时，账户位沿用当前导航展开态的“图标 + 文本”结构，不再额外新发明样式。
- 已登录时直接显示用户名。
- 未登录时直接显示“游客模式”。
- 本轮不增加第二行状态文案，不显示“已登录”“游客模式说明”“账号摘要”等附加信息。

### 2.3 账户位行为

- 已登录时，点击账户位进入个人中心。
- 未登录时，点击账户位进入登录页。
- 焦点流上，账户位就是左侧导航的最后一个可聚焦项。
- 从最后一个导航按钮按下可以进入账户位；从账户位按上返回上一个导航按钮。

### 2.4 常规页面主内容区去壳

- 常规页面右侧主内容区取消外层大圆角深色底板。
- 去掉外层 page shell 的统一深色背景、统一圆角、统一大阴影。
- 页面内容直接落在右侧主舞台上，不再形成“左侧导航 + 右侧大卡片”的双层容器关系。
- 顶部右上角区域不再保留独立 topbar 条。
- 页面本身需要的局部组件层级仍然保留，例如频道切换器、媒体卡片、局部状态块；但不再额外包一个全局外框。

说明：

- 这里的“去掉圆角深色底板”针对的是常规页面主内容区总壳，不是要删除视频卡片、频道按钮等真实业务组件自己的局部表面。
- 播放器沉浸式页面本来就不显示左侧导航和顶部状态栏，本轮不改变播放器全屏壳的基本方向。

### 2.5 播放设置抽屉最终结构

- 播放设置抽屉只保留用户会用到的内容。
- 默认页不重复展示按钮已经表达过的信息。
- 默认页固定为三段：
  - 画质
  - 编码偏好
  - 当前播放结果
- 快捷操作保留在底部，但只放 1 到 2 个真正有用的动作。
- 所有 debug 风格信息全部移出默认页。

## 3. 页面结构调整方案

### 3.1 App Shell

调整后的应用壳结构如下：

1. 左侧：全局导航
2. 左侧底部：账户位
3. 右侧：直接承载页面内容

需要移除的现有结构：

- 顶部 `tv-topbar`
- 右上角 `TopbarProfilePill`
- 常规页面的统一外层深色壳容器视觉

需要保留的结构：

- 左侧导航展开/收起逻辑
- 右侧内容区滚动能力
- 现有页面栈和焦点组划分

### 3.2 左侧导航布局

左侧导航从上到下分成三段：

1. 品牌区
2. 主导航列表
3. 底部账户位

布局要求：

- 主导航列表继续占据可伸缩空间。
- 账户位固定压在底部，不跟导航项混在同一批流式换位里。
- 账户位与主导航列表之间保留明确留白，形成“导航功能”和“账户入口”的分区。
- 展开态下，账户位文本的左边距、字号、图标盒子尺寸与导航项保持同一规则。

### 3.3 常规页面视觉层级

常规页面改成“舞台式内容布局”：

- 背景层：应用整体背景渐变与氛围层保留。
- 内容层：页面标题、频道切换器、泳道、卡片直接放在主内容区里。
- 局部组件层：只有真正需要聚焦或承载信息的组件自己带局部表面。

不再保留：

- 页面最外层的大型深色包裹面板
- 右上角独立账号胶囊
- 为了制造“层级感”而存在的额外边框或圆角大容器

这样处理后的页面应该更接近 TV 端常见结构：左边是稳定导航，右边是直接内容，内容本身决定层级，而不是再靠一层巨大的容器来框住全部页面。

## 4. 播放设置抽屉最终方案

### 4.1 抽屉目标

播放设置抽屉只回答两件事：

- 我现在可以切什么
- 现在实际播成了什么

除此之外的信息默认都不出现。

### 4.2 默认保留的内容结构

默认页固定按下面顺序组织：

1. 画质
2. 编码偏好
3. 当前播放结果
4. 快捷操作（可选，且数量严格控制）

### 4.3 每段具体内容

#### 画质

- 只保留画质按钮组。
- 不再附加长说明文案。
- 选中态直接代表用户当前想要的画质。

#### 编码偏好

- 只保留编码按钮组。
- 选中态直接代表用户当前选择的编码偏好。
- 不再在下方重复写“编码偏好结果”。

#### 当前播放结果

- 只保留一条结果摘要。
- 文案格式固定为：
  - `当前播放：<实际返回画质> / <实际编码> / <当前分辨率>`
- 不拆成多行表格，不再单独列字段名。

#### 快捷操作

- 默认最多保留 2 个动作：
  - 重新加载当前策略
  - 恢复自动策略
- 如果实现后发现操作频率低或视觉仍显拥挤，可以继续缩减为 1 个动作。

### 4.4 本轮移除或下线的默认信息

以下内容从默认抽屉中移出，不再占据主设置页：

- BV 号
- CID
- 请求画质
- 编码偏好的重复状态展示
- 当前执行策略
- 接口限制码
- 当前 Host
- 当前候选序号
- 可切换地址数量
- 视频 MIME
- 音频轨排障明细
- 返回 codec/宣称 codec 等偏调试字段
- 冗长的链路解释性文案
- 候选顺序大段计划文本

这些信息不折叠、不后置，直接从默认抽屉去掉。

### 4.5 文案约束

- 默认设置页不写长句解释。
- 能用按钮选中态表达的，不再写第二遍。
- 当前结果统一写成一条摘要，不做多字段清单。
- 如果必须出现提示，只允许保留 1 条最关键提示。

### 4.6 最终页面示意

最终默认页应接近下面这种密度：

1. 画质按钮组
2. 编码按钮组
3. `当前播放：1080P / AVC / 1920x1080`
4. 1 到 2 个快捷操作按钮

实现时如果超出这个密度，继续删，不再往里加解释性内容。

## 5. 焦点与返回链路要求

### 5.1 左侧导航账户位

- 账户位属于 `nav` 焦点组。
- 行为上等价于左侧导航最后一个 item。
- 不额外发明独立焦点系统。

### 5.2 常规页面

- 页面进入后默认焦点仍然先落在主内容区，而不是左侧导航。
- 只有用户显式按左，才返回导航。
- 去掉顶栏后，焦点不再需要绕过右上角账户胶囊。

### 5.3 播放设置抽屉

- 抽屉打开后继续接管焦点。
- 抽屉关闭后恢复到触发设置的播放器按钮。
- 因为抽屉内容减少，抽屉内部 4-way 导航应更稳定，不再出现用户要穿过大量说明文字才能找到真正操作项的问题。

## 6. 组件与样式层的落地建议

### 6.1 组件层

预计会涉及这些组件的收口或调整：

- `src/components/AppShell.tsx`
- `src/components/SideNavRail.tsx`
- `src/components/TopbarProfilePill.tsx`
- `src/features/player/PlayerSettingsDrawer.tsx`
- `src/features/player/PlayerPage.tsx`

组件方向：

- `TopbarProfilePill` 不再作为 AppShell 顶部入口保留原职责。
- 如需复用其视觉片段，应抽成“可放进导航底部的账户位组件”，而不是继续保留 topbar 语义。
- `SideNavRail` 负责品牌区、主导航区、底部账户位三段式布局。
- `PlayerSettingsDrawer` 负责更轻的结构，不再承担调试控制台的职责。

### 6.2 样式层

预计会涉及这些样式来源：

- `src/styles/app.css`

样式侧重点：

- 删除 `tv-topbar` 相关样式来源
- 删除或弱化常规页面外层 page shell 的统一壳视觉
- 为侧栏底部账户位补齐收起/展开态样式
- 收紧播放器设置抽屉的排版密度与信息结构

## 7. 验收标准

### 7.1 账户位

- 右上角不再出现账号胶囊。
- 左侧导航最底部出现账户位。
- 收起态只显示头像，且与导航按钮图标位对齐。
- 展开态显示头像和用户名，未登录显示“游客模式”。
- 已登录进入个人中心，未登录进入登录页。

### 7.2 常规页面主内容区

- 右侧主内容区不再被一整块大圆角深色底板包住。
- 页面视觉层级比当前明显更轻。
- 不出现新的内容遮挡或焦点不可达问题。

### 7.3 播放设置抽屉

- 默认抽屉只保留画质、编码、1 条当前播放结果和少量快捷操作。
- 不再出现重复状态信息。
- 不再出现长篇调试信息挤占主设置区域的情况。
- 抽屉在 1080p 模拟器和 LG 真机上都能稳定排版，不出现明显文案重叠。

## 8. 实施顺序

建议按下面顺序落地：

1. 先改 `AppShell / SideNavRail`，完成账户位下沉与顶栏移除
2. 再改常规页面主内容区总壳，去掉统一大圆角深色底板
3. 最后重排播放器设置抽屉内容

这样做的原因是：

- 账户入口下沉和主舞台去壳是外层结构调整，优先完成后，页面视觉基线会先稳定下来。
- 播放设置抽屉是局部面板，放在最后处理更容易对照新的整体视觉密度做收口。

## 9. 验证方式

完成实现后至少验证：

- 首页：导航收起/展开、账户位对齐、进入个人中心/登录页
- 其他常规页面：右侧主内容区去壳后是否仍保持清晰可读
- 播放器：打开播放设置后，是否只看到必要设置与状态，且焦点流稳定
- webOS Simulator：1080p 布局是否稳定
- LG C1 真机：账户位、主内容区和播放设置是否与模拟器保持一致

命令验证仍按仓库规则执行：

- `npm run lint`
- `npm run test`
- `npm run typecheck`
- `npm run build`

涉及真机联调时，再补 webOS 打包与部署链路验证。
